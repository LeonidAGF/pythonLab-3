<h2>Соблюдённые требования к коду</h2>
- Переменные, функции и модули именуются по snake_case

- Константы вынесены в файл constants.py и именовановаться с помощию символов в верхнем регистре

- Имена сущностей осмысленные и содержательные

- Все отступы консистентны: 1 TAB = 4 spaces

- Весь функционал описан в функциях.

- К каждой функции описаны docstring и аннотации к аргументам и выходным параметрам функций.

В папке tests лежат unit тесты для проверки функциональности программы или ее частей. Название тестов содержит определение проверямой части. Проверяются не только успешные кейсы, но и краевые условия и кейсы в которых была допущена ошибка (неудачные кейсы).

<h2>Что было необходимо сделать</h2>

    Было необходимо реализовать функции factorial и factorial_recursive, fibo и fibo_recursive, bubble_sort ,и quick_sort ,и counting_sort ,и radix_sort ,и bucket_sort ,и heap_sort, стек или очередь, rand_int_array, reverse_sorted, rand_float_array, timeit_once, benchmark_sorts, поддержку ключей и компараторов в сортировках

<h2>Что было реализованно</h2>

- `factorial` функция для подсчёта факториала
- `factorial_recursive` рекурсивная функция для подсчёта факториала
- `fibo` функция для вычислния числа фибоначи
- `fibo_recursive` рекурсивная функция для вычисления числа фибоначи
- `bubble_sort` пузырьковая сортировка
- `quick_sort` быстрая сортировка
- `counting_sort` сортировка подсчётом
- `radix_sort` поразрядная сортировка
- `bucket_sort` корзинная сортировка
- `heap_sort` пиромидальная сортировка
- `my_queue` очередь построенная на связанном списке
- `rand_int_array` функция генерирующая массив случайных чисел
- `reverse_sorted` функция генерирующая массив чисел от 1 до n идущих в порядке убывания
- `rand_float_array` функция генерирующая массив случайных нецелых чисел
- `timeit_once` функция замеряющая время выполнения другой функции
- `benchmark_sorts` функция возвращающая статистику по разным сортировкам и их скорости выполнения на разных данных
